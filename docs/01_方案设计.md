### 1 设计目标

本系统旨在为中小型制造业企业提供一套通用的内外贸一体化管理解决方案，通过简道云平台实现以下目标：

- **业务流程全覆盖**：整合销售、采购、生产、库存、外贸物流、质量管理等核心业务环节，消除信息孤岛。
- **内外贸一体化**：在统一平台上同时支持国内销售和出口贸易，差异化处理外贸订单的物流、关务、收汇等特殊流程。
- **跟单效率提升**：为内贸/外贸跟单员提供实时订单进度、生产状态、库存信息，支持跨部门高效协同。
- **客户关系可追溯**：记录客户沟通、回访、样品寄送等过程，辅助销售团队维护客户关系。
- **审批流程规范化**：对核心业务单据实施逐级审批，确保业务合规、风险可控。
- **数据驱动决策**：通过仪表盘实时监控关键业务指标，为管理层提供可视化决策依据。

### 2 设计原则

| 原则 | 说明 |
|------|------|
| **聚焦核心** | 优先覆盖销售、采购、生产、库存、外贸物流五大主线，财务仅保留收付款、发票等基础功能，不涉及总账、成本核算等复杂模块。 |
| **内外贸一体** | 销售订单通过“订单类型”字段区分内外销，外贸订单自动触发物流委托、报关、跟踪等子流程，数据模型统一。 |
| **跟单友好** | 为跟单员设计订单进度视图，关联生产、库存、发货信息，支持一键查询和问题定位，减少跨部门沟通成本。 |
| **客户关系管理** | 增加客户跟进记录和样品管理模块，记录销售过程中的关键互动，帮助维护客户关系。 |
| **逐级审批** | 核心业务单据配置多级审批流程，可根据金额、类型等条件动态跳转节点，确保业务合规。 |
| **数据关联** | 所有业务单据通过关联字段建立逻辑链接，实现从客户询盘到最终收汇的全链路追溯。 |
| **可视化优先** | 设计多维度仪表盘，将分散的业务数据转化为直观的图表和指标，辅助管理层决策。 |
| **模块化设计** | 划分为主数据、技术管理、销售、采购、生产、质量、库存、外贸物流、财务等9个模块，便于扩展和维护。 |
| **通用性** | 方案不绑定任何具体行业或产品，企业可根据自身实际情况自定义基础数据（如部门、职位、产品等）。 |

### 3 组织架构

本系统面向典型中小型制造业企业，建议组织架构如下：

```
总经理

├─ 生产系统
│   └─ 生产部：生产总监、生产经理、生产主管、班组长、操作工
├─ 质量系统
│   └─ 质检部：质检经理、质检主管、质检员
├─ 供应链系统
│   ├─ 采购部：采购经理、采购主管、采购专员
│   └─ 仓储物流部：仓储经理、仓库管理员、物流专员、单证员
├─ 市场系统
│   └─ 销售部：销售总监、销售经理、销售专员、跟单员（内贸/外贸）
├─ 技术系统
│   └─ 研发部：研发总监、研发工程师、技术员
└─ 职能系统
    ├─ 总经办：总经理
    ├─ 财务部：财务经理、会计、出纳
    ├─ 行政部：行政经理、行政专员
    └─ 人事部：人事经理、人事专员
```

**角色职责说明**：

| 角色 | 所属部门 | 核心职责 |
|------|----------|----------|
| 销售专员（业务员） | 销售部 | 开发客户、报价、谈判、签订销售订单 |
| 跟单员（内贸/外贸） | 销售部 | 负责订单执行全过程跟踪，包括生产进度、物流协调、客户沟通、单证制作等 |
| 物流专员 | 仓储物流部 | 负责订舱、拖车、货代对接、物流跟踪 |
| 单证员 | 仓储物流部 | 负责制作报关资料、管理提单等单证 |
| 生产计划员 | 生产部 | 负责根据销售订单或预测创建生产工单，监控物料齐套性 |

### 6 核心业务流程

#### 6.1 销售流（内外贸）
- **内销**：客户询盘 → 销售报价 → 销售订单(内销) → 生产/备货 → 发货单 → 销售出库(扣库存) → 销项发票 → 收款记录
- **外贸**：客户询盘 → 销售报价 → 销售订单(外贸) → 物流委托单 → 订舱/拖车/报关 → 获取提单 → 销售出库 → 销项发票(形式发票) → 收汇记录

#### 6.2 采购流
- 采购申请 → 采购订单 → 供应商发货 → 采购入库(质检合格) → 进项发票 → 付款记录
- 采购退货：采购退货单 → 退货出库

#### 6.3 生产流
- 销售订单/计划 → 生产工单 → 生产领料(出库) → 工序报工 → 质量检验(成品) → 生产入库 → 生产退料(余料退回)

#### 6.4 仓储流
- 所有出入库均通过出入库流水表记录，实时更新库存台账。
- 库存调拨、盘点等操作独立处理。

#### 6.5 外贸物流流
- 外贸销售订单 → 物流委托单 → 订舱 → 装箱 → 报关 → 出运 → 提单 → 物流跟踪

### 7 审批流程设计（简化版）

| 表单 | 审批节点 | 条件分支 | 说明 |
|------|----------|----------|------|
| 销售订单 | 提交→销售经理审核 | 金额≥50万元需总经理终审 | 取消财务复核环节，简化流程 |
| 采购订单 | 提交→采购经理审核 | 金额≥30万元需总经理终审 | 由采购经理直接判断 |
| 生产工单 | 提交→生产主管审核 | 无分支 | 仅需生产主管确认 |
| 工程变更 | 提交→技术经理审核 | 重大变更需总经理批准 | 重大变更指涉及核心部件更换 |
| 质量检验 | 录入→质检主管复核 | 仅不合格品需复核 | 合格品自动关闭 |
| 物流委托单 | 提交→仓储经理审核 | 费用≥5万元需总经理 | |

**注**：审批阈值可根据企业实际规模调整。

### 8 内外贸差异对比

| 维度 | 内销 | 外贸 |
|------|------|------|
| 销售订单字段 | 常规字段 | 增加贸易术语、目的港、货代要求、唛头、信用证号 |
| 发货流程 | 发货单 + 国内物流 | 物流委托单 + 订舱/报关/提单 |
| 物流跟踪 | 国内运单号 | 物流跟踪表（国际节点：ETD/ETA、提单号） |
| 财务处理 | 增值税发票 + 国内收款 | 形式发票(Commercial Invoice) + 收汇 + 出口退税 |
| 跟单重点 | 协调仓库发货 | 制作单证、跟踪船期、协调报关 |

### 12 关键数据关联关系

- **销售订单** ↔ 客户表、产品表（通过明细表）
- **销售订单(外贸)** ↔ 物流委托单（一对多）
- **物流委托单** ↔ 物流跟踪表（一对多）、报关资料表（一对一）
- **采购订单** ↔ 供应商表、产品表（通过明细表）
- **采购订单** ↔ 采购入库单、进项发票、付款记录
- **生产工单** ↔ 销售订单（可选）、产品表
- **生产工单** ↔ 工序报工表、生产领料单、生产入库单
- **库存台账** ↔ 出入库流水表
- **收款记录** ↔ 销售订单；**付款记录** ↔ 采购订单
- **发票** ↔ 销售订单/采购订单
- **客户跟进记录** ↔ 客户表、销售订单（可选）
- **样品管理表** ↔ 客户表、产品表
- **员工表** ↔ 部门表、班组表、职位表（通过联动实现）

### 13 员工表多级联动设置

- **部门表、班组表、职位表**作为辅助数据，需预先导入。
- 在员工表中：
  - **所属部门**：关联部门表，显示部门名称。
  - **职位**：关联职位表，设置过滤条件：`职位表.所属部门 = 员工表.所属部门`。
  - **所属班组**：关联班组表，设置过滤条件：`班组表.所属部门 = 员工表.所属部门`。
  - **字段显隐规则**：仅当`所属部门 = 生产部`且`职位 = 班组长或操作工`时，显示“所属班组”字段。

### 14 期初设置

- **期初库存**：通过导入库存台账，同时生成一条“期初导入”类型的出入库流水，作为初始余额。
- **客户/供应商期初余额**：可通过收款/付款记录表手动录入期初应收/应付，或通过仪表盘视图手工初始化。
- **仓库仓位**：在仓库仓位表中预先维护所有仓库及仓位信息。

### 15 方案解决的核心痛点

1. **信息孤岛问题**：各部门数据分散，无法协同。通过统一平台和关联字段，实现数据互通，订单状态实时共享。
2. **内外贸流程割裂**：将内外贸订单整合于同一张销售订单表，通过订单类型和条件字段差异化处理，流程统一且灵活。
3. **审批不规范、风险不可控**：对关键业务单据配置逐级审批流程，支持金额阈值跳转，确保合规。
4. **库存账实不符**：通过出入库流水表联动库存台账，所有业务单据自动或手动产生流水，确保库存动态准确，并设置安全库存预警。
5. **生产进度难跟踪**：通过工序报工实时反馈进度，完工质检后自动入库，工单状态一目了然。
6. **外贸物流不可视**：增加物流委托单和物流跟踪表，将外贸物流纳入统一管理，提单号、ETD/ETA全程可查。
7. **跟单效率低**：为跟单员提供订单进度看板，实时掌握生产、发货状态，减少沟通成本。
8. **客户跟进无记录**：通过客户跟进记录表和样品管理表，沉淀销售过程信息，便于团队协作。
9. **决策缺乏数据支撑**：设计多维度仪表盘，实时展示销售、生产、库存、采购、物流、财务等关键指标，支持数据驱动决策。

### 16 逻辑自洽性审查总结

经全面审查，本方案的核心业务逻辑、数据关联及模块顺序均符合企业管理实际，无逻辑矛盾或依赖缺失：

- **模块顺序**：基础数据→技术管理→销售→采购→生产→质量→库存→外贸物流→财务，满足所有依赖关系。
- **时序约束**：各表单内日期字段均已确保业务时序（如保养日≥购入日、承诺交期≤要求交期、入库日期≥下单日期等）。
- **数据关联**：所有跨表单引用均采用唯一编码，符合简道云关联规范。
- **内外贸差异**：销售订单通过“订单类型”区分，并触发不同子流程，逻辑清晰。
- **跟单员支持**：仪表盘设计覆盖跟单员核心工作流，可显著提升效率。

本方案通用性强，不绑定任何特定行业或产品，企业可根据自身实际情况自定义基础数据，灵活适配。
