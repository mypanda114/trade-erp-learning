### 1 脚本体系概述

本系统由 10 个 Python 脚本组成，其中 9 个模块脚本对应 9 个业务模块，1 个主控脚本负责统一调用。所有脚本均以数字前缀命名，与设计方案中的模块顺序一致。

| 序号 | 脚本文件 | 模块名称 | 生成表单数量 |
|------|----------|----------|--------------|
| 1 | `1_gen_base.py` | 基础数据模块 | 9 |
| 2 | `2_gen_plm.py` | 技术管理模块 | 3 |
| 3 | `3_gen_sales.py` | 销售管理模块 | 8 |
| 4 | `4_gen_purchase.py` | 采购管理模块 | 5 |
| 5 | `5_gen_production.py` | 生产管理模块 | 5 |
| 6 | `6_gen_quality.py` | 质量管理模块 | 1 |
| 7 | `7_gen_inventory.py` | 库存管理模块 | 4 |
| 8 | `8_gen_logistics.py` | 外贸物流模块 | 3 |
| 9 | `9_gen_finance.py` | 财务管理模块 | 4 |
| 10 | `generate_all.py` | 主控脚本 | - |

### 2 从零开始搭建本地环境

#### 2.1 创建项目目录

打开终端（PowerShell / CMD / bash），执行以下命令：

```powershell
# 在 D 盘创建项目文件夹（可根据需要修改路径）
cd D:\
mkdir Trade-ERP-Learning
cd Trade-ERP-Learning

# 创建子文件夹
mkdir docs, src, output, examples -Force
```

最终目录结构如下：

```
D:\Trade-ERP-Learning
├─ docs          # 存放所有文档（Markdown）
├─ src           # 存放 Python 脚本
├─ output        # 存放生成的 Excel 文件（后续可被 .gitignore 忽略）
└─ examples      # 可选，存放示例数据或模板
```

#### 2.2 初始化 Git（可选）

如需版本控制，可执行：

```powershell
git init
echo "output/" > .gitignore
echo "__pycache__/" >> .gitignore
echo ".venv/" >> .gitignore
git add .
git commit -m "初始化项目结构"
```

#### 2.3 创建 Python 虚拟环境

在项目根目录执行：

```powershell
python -m venv .venv
```

激活虚拟环境：

- **Windows PowerShell**：`.\.venv\Scripts\Activate.ps1`
- **Windows CMD**：`.venv\Scripts\activate.bat`
- **macOS/Linux**：`source .venv/bin/activate`

激活后终端提示符前会出现 `(.venv)`。

#### 2.4 安装依赖库

确保虚拟环境已激活，执行：

```powershell
pip install pandas openpyxl
```

### 4 脚本功能与设计原则

#### 4.1 通用设计原则

- **独立运行**：每个脚本都可单独执行，自动创建对应输出文件夹并生成文件。
- **随机种子**：默认固定为 `random.seed(42)`，保证结果可复现；注释掉该行可生成不同数据。
- **文件命名**：生成的 Excel 文件直接带有数字前缀（如 `1.部门表.xlsx`），与设计方案顺序一致。
- **路径处理**：所有输出路径均使用相对于项目根目录的绝对路径（`D:\Trade-ERP-Learning\output`），请根据实际情况调整。

#### 4.2 核心业务规则

- **员工筛选**：所有涉及员工的字段（报价人、销售专员、跟单员、申请人、操作人、检验员等）均严格遵循：
  - 部门归属：根据部门名称筛选（如销售部、采购部等）。
  - 在职状态：仅选择状态为“在职”的员工。
  - 入职日期约束：业务单据日期 ≥ 员工入职日期。
- **日期时序**：各表单内日期字段已确保业务合理性（详见 [02_表单详细设计](02_表单详细设计.md)）。
- **关联字段**：所有跨表单引用均使用唯一编码（如客户编码、产品编码），便于简道云中设置关联。

#### 4.3 模块依赖关系

| 模块 | 前置依赖 | 说明 |
|------|----------|------|
| 基础数据 | 无 | 必须先于其他模块运行 |
| 技术管理 | 产品、员工 | 引用产品表和员工表 |
| 销售管理 | 客户、产品、员工 | 完全依赖基础数据 |
| 采购管理 | 供应商、产品、员工 | 完全依赖基础数据，可选关联销售订单 |
| 生产管理 | 产品、员工、设备 | 可独立运行（BOM/工艺路线硬编码） |
| 质量管理 | 产品、员工、采购/生产入库 | 需读取前序单据（若无则用模拟单号） |
| 库存管理 | 产品、仓库、员工 | 可独立模拟流水 |
| 外贸物流 | 销售订单（外贸） | 需销售模块先运行 |
| 财务管理 | 销售订单、采购订单 | 需销售和采购模块先运行 |

### 5 运行脚本生成数据

#### 5.1 单独生成基础数据（可选）

```powershell
cd src
python 1_gen_base.py
```

执行后，`output\1.基础数据` 文件夹中应出现 9 个带数字前缀的 Excel 文件。

#### 5.2 生成全部数据（推荐）

```powershell
python generate_all.py
```

主控脚本将依次调用所有模块脚本，并在 `output` 下创建 9 个模块文件夹，每个文件夹内包含对应表单（已按设计方案顺序命名）。

#### 5.3 验证数据

您可以打开任意一个生成的 Excel 文件，检查字段是否完整，数据是否符合业务逻辑（例如销售报价单中的报价人应来自销售部且状态为在职）。也可使用简单的 Python 脚本进行批量验证（参考 [02_表单详细设计](02_表单详细设计.md) 中的验证思路）。

### 6 脚本维护与扩展

- **修改数据量**：在对应模块脚本中调整循环范围（如销售订单数量、员工人数等）。
- **调整业务规则**：修改员工筛选条件、日期生成逻辑等。
- **增加新模块**：按现有模板编写新脚本，并添加到主控脚本的调用列表中。
- **代码结构**：每个脚本均包含清晰的注释和函数定义，便于二次开发。

---

### 7 注意事项

- 运行主控脚本前，建议备份 `output` 目录，以免覆盖重要数据。
- 若单独运行业务模块，需确保基础数据已生成（先运行 `1_gen_base.py`）。
- 所有脚本均依赖正确的文件路径，请确保 `output` 目录与脚本中的相对路径一致（默认使用 `../output/` 相对于 `src` 的位置）。
- 生成的 Excel 文件可直接用于导入简道云，具体操作请参考 [04_简道云搭建指南](04_简道云搭建指南.md)。
